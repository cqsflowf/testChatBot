# 智能体应用产品需求文档（PRD）

## 1. 产品概述

### 1.1 产品定位
打造一款类似豆包的智能对话助手，支持多模态输入、快速响应、高质量回答，通过 Thinker-Talker 双模型协作架构实现流畅自然的交互体验。

### 1.2 目标用户
- 需要快速获取信息的知识工作者
- 需要行程规划（酒店、景点、机票）的旅行者
- 需要实时资讯查询的普通用户
- 需要多模态交互的创意工作者

### 1.3 核心价值
1. **极速响应**：首字响应时间 < 1s，用户感知无延迟
2. **高质量回答**：结合实时信息与深度推理，提供准确、有深度的回答
3. **自然交互**：支持图文混合输入，智能理解用户意图
4. **专业工具**：天气查询、联网搜索、旅行信息聚合

### 2.3 Thinker-Talker 双模型架构

#### 2.3.1 Talker（快响应模型）
- **定位**：面向用户的前端响应模型
- **职责**：
  - 快速生成初步回复（< 1s）
  - 维持对话流畅性和自然度
  - 简单问题的直接回答
  - 情感化、人格化的表达
- **特点**：
  - 轻量级、高速度
  - 优秀的对话能力
  - 适合即时交互

#### 2.3.2 Thinker（深度思考模型）
- **定位**：后台深度推理模型
- **职责**：
  - 复杂任务的深度规划
  - 工具调用的决策与执行
  - 多步骤推理任务
  - 回答质量的精修与补充
- **特点**：
  - 强推理能力
  - 工具调用能力
  - 适合复杂任务

#### 2.3.3 响应融合策略
```
融合原则：
1. Talker 输出为主基调，保证响应速度和流畅度
2. Thinker 补充详细内容，不破坏 Talker 的逻辑结构
3. 融合时机：
   - Talker 先输出开场白/初步答案
   - Thinker 完成推理后，以"补充"/"更正"形式追加
4. 冲突处理：Thinker 可修正 Talker 的错误，但需温和过渡
```

**融合示例**：
```
[Talker 快速响应，0.8s]
"北京今天天气不错，气温在 15-22 度之间。"

[Thinker 补充，2.5s]
"根据最新数据，今天北京空气质量为优（AQI: 35），
适合户外活动。另外，下午 3 点后可能有微风，
出门可以带件薄外套。"
```

## 5. 用户交互流程

### 5.1 简单问答流程
```
用户输入 → 意图识别 → Talker 直接回答 → 流式输出
     ↓
   [0.8s 内开始响应]
```

### 5.2 工具调用流程
```
用户输入 → 意图识别 → Thinker 规划
     ↓
并行执行：
  ├─ Talker 先行响应（"让我帮你查一下..."）
  └─ Thinker 调用工具 → 获取结果 → 生成详细回答
     ↓
内容融合 → 流式输出完整回答
     ↓
   [1s 内开始响应，3s 内完成]
```

### 5.3 多模态输入流程
```
用户上传图片+文本
     ↓
多模态编码器 → 图文特征融合
     ↓
意图分类：
  ├─ 图片理解类 → Talker + 视觉模型
  ├─ 搜索类 → Thinker + 搜索工具
  └─ 混合类 → Talker 先行 + Thinker 深度处理
     ↓
响应融合 → 流式输出
```

## 10. 对话中断与意图管理

### 10.1 中断检测机制

| 场景 | 检测方式 | 处理策略 |
|------|----------|----------|
| 用户主动中断 | 用户发送新消息 | 暂停当前任务，处理新请求 |
| 意图切换 | 新消息意图与当前任务不同 | 保存上下文，切换到新意图 |
| 任务取消 | 用户明确表示取消 | 终止任务，清理资源 |
| 超时中断 | 任务执行超时 | 返回部分结果，提示用户 |

### 10.2 意图冲突判断规则

```
冲突判断流程：
1. 解析新消息意图
2. 对比当前执行任务的意图
3. 判断冲突类型：
   - 完全无关：暂停当前，处理新请求
   - 部分相关：尝试合并处理
   - 补充信息：融入当前任务
   - 取消指令：终止当前任务
```

| 冲突类型 | 示例 | 处理方式 |
|----------|------|----------|
| 完全无关 | 查天气时问"今天股市怎么样" | 暂停天气查询，处理股市问题 |
| 部分相关 | 查北京天气时说"还有上海的" | 并行查询两个城市 |
| 补充信息 | 查酒店时说"要五星级的" | 更新查询条件继续执行 |
| 取消指令 | "算了不查了" | 终止任务，确认取消 |

### 10.3 上下文保持策略

| 策略 | 描述 | 适用场景 |
|------|------|----------|
| 完整保持 | 保留所有上下文信息 | 短暂中断后恢复 |
| 摘要保持 | 保留关键信息摘要 | 长时间中断后恢复 |
| 选择性保持 | 仅保留相关上下文 | 意图切换时 |
| 清空重置 | 清除所有上下文 | 用户明确要求重新开始 |

### 10.4 任务状态管理

```
任务状态机：
┌─────────┐    开始    ┌───────────┐    完成    ┌──────────┐
│ pending │ ─────────► │ executing │ ─────────► │ completed│
└─────────┘            └───────────┘            └──────────┘
                            │
                     中断/暂停│
                            ▼
                      ┌───────────┐    恢复    ┌───────────┐
                      │  paused   │ ─────────► │ executing │
                      └───────────┘            └───────────┘
                            │
                       取消 │
                            ▼
                      ┌───────────┐
                      │ cancelled │
                      └───────────┘
```

| 状态 | 描述 | 可转换状态 |
|------|------|------------|
| pending | 等待执行 | executing, cancelled |
| executing | 正在执行 | completed, paused, cancelled |
| paused | 已暂停 | executing, cancelled |
| completed | 已完成 | - |
| cancelled | 已取消 | - |
