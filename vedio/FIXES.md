# 全双工语音对话系统 v2.0 - 修复说明

## 问题诊断

原始代码存在以下问题：
1. **语音识别返回固定文本**：`qwen_omni.py` 中的模拟模式无法真正识别语音，始终返回固定文本
2. **环境变量未加载**：配置模块没有自动加载 `.env` 文件，导致 API Key 无法读取
3. **多轮对话历史未正确维护**：对话历史更新时机有问题

## 修复内容

### 1. 语音识别修复 (`qwen_omni.py`)
- 使用 Qwen-Audio 模型进行真实语音识别
- 正确传递音频数据（PCM 格式，base64 编码）
- 返回实际识别的文本内容

### 2. 配置加载修复 (`config.py`)
- 添加 `python-dotenv` 自动加载 `.env` 文件
- 新增 ASR 配置属性

### 3. 核心系统修复 (`system.py`)
- 完善语音处理流程
- 正确更新对话历史
- 添加详细日志输出

### 4. 新增模块
- `realtime_asr.py`: 实时语音识别模块
- `test_e2e.py`: 端到端测试用例

## 使用方法

### 1. 配置 API Key
```bash
# 编辑 .env 文件
DASHSCOPE_API_KEY=your-api-key-here
```

### 2. 启动服务器
```bash
python start.py
```

### 3. 访问 Web 界面
打开浏览器访问: http://localhost:8765

## 架构说明

```
用户语音 -> 声学VAD -> 音频缓冲 -> Qwen-Audio ASR -> 文本
                                                          |
                                                          v
        TTS <- LLM响应 <- 工具调用 <- 意图识别 <- 情绪识别
         |
         v
      音频输出
```

## 注意事项

1. **语音识别**：需要有效的 DashScope API Key
2. **TTS**：默认使用 Edge TTS（免费），无需 API Key
3. **多轮对话**：系统自动维护最近 10 轮对话历史
4. **打断功能**：说话时可以随时打断

## 常见问题

### Q: 语音识别返回"[模拟模式...]"
A: 请检查 `.env` 文件中的 `DASHSCOPE_API_KEY` 是否正确配置

### Q: WebSocket 连接失败
A: 确保服务器已启动，端口 8765 未被占用

### Q: 工具调用失败
A: 检查日志输出，确认 LLM 正确识别了意图
