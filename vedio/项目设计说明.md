## 一、项目概述

### 1.1 项目简介

全双工语音对话系统是一个支持实时语音交互的智能对话系统，具备以下核心能力：

- **流式语音识别**：使用 Qwen ASR 17B 进行实时语音转文字，帧长20ms
- **流式语义判断**：使用 Qwen-flash 边说边判断用户是否说完
- **有效人声判断**：区分语气助词和有效语义内容，精准触发打断
- **并行情绪识别**：使用 Qwen3-omni-flash 音频输入能力实时分析用户情绪
- **语义驱动打断**：判断到有效人声立即停止TTS，不等声学静音阈值
- **THINKING状态打断**：支持在LLM推理期间打断，取消LLM请求
- **并发音频处理**：音频接收与LLM推理解耦，支持真正的全双工交互
- **LLM流式输出**：LLM响应实时流式显示，用户无需等待完整响应
- **流式TTS播报**：使用 Qwen3-tts-flash 按句子实时转换语音，边生成边播报
- **多轮对话**：支持上下文连贯的多轮对话
- **工具调用**：支持天气查询、时间查询、设备控制等工具
- **时延监控**：实时追踪各模块处理时延，便于性能分析

```python
    语义VAD流式处理器 v3.2

    1. 语义完整性判断（说完/继续说/拒识）
    2. 有效人声判断（区分语气助词和有效内容）
```

#### 2.2.3 判断逻辑

```python
def check_voice_validity(self, text: str) -> str:
    # 1. 检查是否有有效人声关键词
    for word in self.VALID_VOICE_WORDS:
        if word in text:
            return VoiceValidity.VALID

    # 2. 去除语气助词后的文本
    clean_text = text
    for filler in self.FILLER_WORDS:
        clean_text = clean_text.replace(filler, "")
    clean_text = clean_text.strip()

    # 3. 如果有足够有效内容，认为是有效人声
    if len(clean_text) >= 2:
        return VoiceValidity.VALID

    # 4. 只有语气助词
    if text and not clean_text:
        return VoiceValidity.FILLER

    return VoiceValidity.PENDING
```

```python
class StreamingTTSProcessor:
    """
    流式TTS处理器 v3.3

    核心功能：
    - 接收LLM流式输出的文本块
    - 按句子分段转换
    - 每个句子完成后立即发送音频（流畅播报）
    - 使用异步锁确保先进先出顺序

    分段策略：
    - 遇到句子结束符（。！？等）时，立即转换这个句子并发送
    - 确保每个音频块是完整句子，播放更自然

    顺序保证：
    - 使用 asyncio.Lock() 确保文本按顺序处理
    - 先进先出队列，避免音频乱序
    """

    # 句子结束符
    SENTENCE_ENDINGS = ['。', '！', '？', '；', '\n']

    # 配置参数
    MIN_CHUNK_SIZE = 5       # 最小分段长度（字符）
    MAX_CHUNK_SIZE = 80      # 最大分段长度（字符）

    async def add_text(self, text: str) -> bool:
        """添加LLM输出的文本块（使用锁确保顺序）"""

    async def finalize(self) -> bytes:
        """完成处理，返回所有音频数据"""
```

**流式TTS工作流程：**

```
LLM流式输出文本
       │
       ▼
StreamingTTSProcessor.add_text()
       │
       └── 获取异步锁 → 累积文本 → 检测句子结束符？
              │
              ├── 是 → 立即TTS转换 → 发送音频 → 释放锁
              │
              └── 否（达到80字符）→ TTS转换 → 发送音频 → 释放锁
```

**顺序保证机制：**

```
LLM输出文本块1 → add_text() → 获取锁 → 转换句子1 → 发送音频1 → 释放锁
LLM输出文本块2 → add_text() → 等待锁 → 获取锁 → 转换句子2 → 发送音频2
LLM输出文本块3 → add_text() → 等待锁 → 获取锁 → 转换句子3 → 发送音频3
```

```python
async def process_audio(self, audio_chunk: bytes) -> Optional[DialogResult]:
    """
    处理音频块 - v3.2 语义驱动打断

        await self._process_audio_parallel(audio_chunk)

        # 检查是否确认打断（有效人声）
        interrupt_result = self._check_interrupt_voice_validity()

        if interrupt_result == "valid":

        elif interrupt_result == "complete":
            # 语义完整，结束打断确认，进入LLM处理
            return await self._finalize_interrupt_to_llm()

        elif interrupt_result == "timeout":
            # 超时处理
            ...

    # 2. SPEAKING状态下检测打断
    if self.dialog_state.state == DialogState.SPEAKING:
        if self.acoustic_vad.check_interrupt(audio_chunk):
            # 启动打断确认模式
```

#### 2.6.2 打断确认检查

```python
def _check_interrupt_voice_validity(self) -> str:
    """
    检查打断确认结果

    Returns:
        "valid" - 确认是有效人声（应停止TTS）
        "complete" - 语义完整（应进入LLM处理）
        "pending" - 仍在判断中
        "timeout" - 超时
    """
    text = self._asr_text_buffer.strip()

    # 超时检查
    elapsed = time.time() * 1000 - self._interrupt_start_time
    if elapsed > self.INTERRUPT_CONFIRM_TIMEOUT_MS:
        return "timeout"

    # 使用语义VAD判断人声有效性
    voice_validity = self.semantic_vad.processor.check_voice_validity(text)

    if voice_validity == VoiceValidity.VALID:
        # 是有效人声，检查是否语义完整
        if self.semantic_vad.processor.is_complete():
            return "complete"
        return "valid"

    elif voice_validity == VoiceValidity.FILLER:
        # 只是语气助词，继续等待
        return "pending"

    return "pending"
```

```python
async def _process_with_llm(self, text, ...):
    """融合阶段：文本+情绪 → LLM处理（流式显示 + 按句子TTS播报）"""

    # 清空旧音频，准备新问题播报
    await self._notify_clear_audio()

    # 创建流式TTS处理器

    # 定义LLM文本块回调：发送给前端显示 + TTS处理器

        on_tool_detected=on_tool_detected

    full_audio = await self._streaming_tts.finalize()
```

```
│       │                                                         │
│       ▼                                                         │
│  音频放入队列 (非阻塞)                                           │
│       │                                                         │
│       ├──────────────────────────────────────┐                  │
│       ▼                                      ▼                  │
│  音频处理后台任务                        LLM 后台任务            │
│  (持续从队列取音频处理)                (独立运行)                │
│       │                                      │                  │
│       ├── 检测打断 ────────────────────► 取消 LLM 任务          │
│       │                                      │                  │
│       ▼                                      ▼                  │
│  声学VAD/ASR/语义VAD                    流式输出/TTS             │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**关键代码实现**：

```python
# process_audio() - 非阻塞接收
async def process_audio(self, audio_chunk: bytes):
    # 启动后台音频处理任务
    if self._audio_processor_task is None or self._audio_processor_task.done():
        self._audio_processor_task = asyncio.create_task(self._audio_processing_loop())

    # 将音频放入队列（非阻塞）
    self._audio_queue.put_nowait(audio_chunk)
    return None  # 立即返回，不阻塞

# LLM 作为后台任务运行
self._llm_task = asyncio.create_task(self._process_with_llm(...))
```

### 3.2 打断流程图

```
┌─────────────────────────────────────────────────────────────────┐
│                    v3.5 打断流程 (真正全双工)                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  WebSocket 主循环持续接收音频                                    │
│       │                                                         │
│       ▼                                                         │
│  音频处理后台任务检测到人声                                       │
│  (无论当前是什么状态)                                            │
│       │                                                         │
│       ├──── SPEAKING状态 ──→ 停止TTS，启动打断确认               │
│       │                                                         │
│       ├──── THINKING状态 ──→ 取消LLM任务，启动打断确认           │
│       │                                                         │
│       ▼                                                         │
│  启动打断确认模式                                                │
│  - 启动ASR流式识别                                              │
│  - 启动语义VAD（打断模式）                                       │
│       │                                                         │
│       ▼                                                         │
│  语义VAD判断人声有效性                                          │
│       │                                                         │
│       ├──── FILLER(语气助词) ──→ 取消打断，继续播报              │
│       │                                                         │
│       ├──── VALID(有效人声) ──→ 确认打断，继续接收用户输入       │
│       │                                                         │
│       └──── COMPLETE(语义完整) ──→ 创建新LLM后台任务处理         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3.3 可打断状态

| 状态 | 是否可打断 | 说明 |
|------|-----------|------|
| SPEAKING | ✅ | 停止TTS播报，继续接收用户输入 |
| THINKING | ✅ | 取消LLM请求，停止TTS处理器 |
| PROCESSING | ❌ | 时间窗口很短（~100ms），不检测打断 |
| LISTENING | ❌ | 正在监听用户输入，不需要打断 |
| IDLE | ❌ | 空闲状态 |

### 3.3 关键参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 打断确认超时 | 1500ms | 最长等待语义判断时间 |
| 声学VAD静音阈值 | 500ms | Silero VAD静音检测阈值 |
| 最大静音等待 | 2000ms | 超时保护 |
